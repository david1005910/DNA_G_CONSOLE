# DNA_G_OS (v9.0 Core)

## 1. 프로젝트 개요

**시스템 명칭**: DNA_G_OS (바이러스 게놈 수집 및 분석 콘솔)  
**목표**: 글로벌 데이터베이스(NCBI)로부터 실제 바이러스 유전 데이터를 자동으로 수집하고, 머신러닝을 통해 서열 패턴을 분석하며, 특정 유전적 모티프를 기반으로 데이터를 분류합니다.

---

## 2. 데이터 수집 분석 (입력 데이터)

본 시스템은 단순히 무작위 서열을 생성하는 것이 아니라, **NCBI Nucleotide Database** (미국 국립생물공학정보센터)에서 실제 생물학적 데이터를 추출합니다.

### 대상 데이터 사양

시스템은 현재 아래와 같은 엄격한 바이오 필터를 통해 **바이러스 게놈**을 수집하도록 설정되어 있습니다:

| 매개변수 | DNA 설정 | RNA 설정 |
| :--- | :--- | :--- |
| **대상 생물 (Organism)** | `Viruses` (모든 바이러스 과) | `Viruses` (모든 바이러스 과) |
| **분자 유형 (Molecule Type)** | `biomol_genomic` (유전체 DNA) | `biomol_mrna` (메신저 RNA) |
| **서열 길이 (Length)** | `200` ~ `1000` 염기쌍 (bp) | `200` ~ `1000` 염기쌍 (bp) |

**의미**:  
현재 시스템은 바이러스 식별에 주로 사용되는 특정 유전자 또는 부분 게놈 조각에 해당하는 200~1000bp 길이의 기능적 바이러스 세그먼트 저장소를 구축하고 있습니다.

## 3. 머신러닝 및 임베딩 로직

### 여기서 "임베딩"이란 무엇인가?

본 프로젝트에서 "임베딩"은 **K-mer 빈도 벡터화 (K-mer Frequency Vectorization)**, 구체적으로는 3-gram 방식을 의미합니다.

1. **원본 입력**: `AGCTGCGCTA...` (생물학적 서열)
2. **토큰화 (Tokenization)**: 시스템은 서열을 서로 겹치는 3글자 단위 단어(코돈/모티프)로 분해합니다.
   * 예: `AGC`, `GCT`, `CTG`, `TGC`...
3. **벡터화 (Vectorization)**: 가능한 모든 3글자 조합이 서열 내에 몇 번 등장하는지 카운트합니다.
   * 이 과정은 생물학적 문자열을 컴퓨터가 계산할 수 있는 **수학적 벡터**로 변환합니다.
   * *비유*: 책에서 "그", "그리고", "사과" 같은 단어들이 몇 번 나오는지 세어서 그 책의 장르를 추측하는 것과 같습니다.

### 현재 분류 로직

AI 모델(`RandomForestClassifier`)은 현재 **"GCG-Rich" 서명**을 탐지하도록 학습되어 있습니다:

* **Type A (고밀도)**: `GCG` 패턴이 **5회 이상** 포함된 서열.
  * *생물학적 의미*: 높은 GC 함량 영역은 주로 안정적인 유전자 구조나 특정 바이러스 클래스(예: 헤르페스 바이러스)와 연관이 깊습니다.
* **Type B (저밀도)**: `GCG` 패턴이 적게 등장하는 서열.
  * *생물학적 의미*: AT가 풍부한 영역은 다른 유형의 바이러스(예: 일부 플라비 바이러스)나 유전자 사이 영역(intergenic regions)의 일반적인 특징입니다.

> **실제 응용**: 이 로직은 게놈 내에서 "CpG 섬(CpG Islands)"이나 특정 단백질 결합 부위를 찾는 과정을 시뮬레이션한 것입니다.

## 4. 시스템 기능 및 확장성

### 이 시스템으로 무엇을 할 수 있는가?

1. **바이러스 감시**: 설정된 기준에 맞는 새로운 바이러스 데이터가 NCBI에 등록되는지 실시간으로 모니터링하고 자동 수집합니다.
2. **패턴 발견**: "Feature Sensitivity Analytics" 패널을 통해 수집된 바이러스를 구분하는 데 어떤 3-base 패턴(예: `AAA`, `GCG`)이 가장 중요한지 시각적으로 확인할 수 있습니다.
3. **변종 탐지 (Anomaly Detection)**: 수집된 데이터를 바탕으로 모델을 재학습시키면, 시스템은 현재 데이터셋의 "정상 패턴"을 정교하게 학습합니다. 향후 이 패턴에서 크게 벗어나는 새로운 서열이 수집되면 이를 변종으로 분류하거나 보고할 수 있습니다.

### 기술적 구성

* **백엔드**: Python 3.9 + Flask
* **데이터 원천**: NCBI E-utilities (E-search, E-fetch) REST API
* **데이터베이스**: SQLite (로컬 고속 저장소)
* **AI 엔진**: Scikit-Learn (Random Forest + CountVectorizer)
* **프론트엔드**: React + Vite (사이버펑크 대시보드 인터페이스)

## 5. 요약

당신은 현재 **실시간 바이러스 바이오 감시 터미널**을 가동 중입니다. 이 시스템은 미국 국립의학도서관(NLM)으로부터 실제 유전 코드를 가져와 3-base 패턴 기반의 수학적 벡터로 분해하며, GCG 모티프 밀도에 따라 분류함으로써 고유한 바이러스 시그니처를 식별하는 과정을 시뮬레이션합니다.
